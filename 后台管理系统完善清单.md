### 10. 内容审核系统 ❌
**状态**: 未实现
**优先级**: 高

**需要实现的功能**:
- 敏感词管理（增删改查）
- 审核规则配置
- 待审核内容队列（动态、评论、聊天消息）
- 审核历史记录
- 审核统计报表
- 自动审核开关
- 审核强度配置（宽松/标准/严格）

**数据库设计**:
```sql
-- 敏感词表
CREATE TABLE IF NOT EXISTS sensitive_words (
    id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    word VARCHAR(50) NOT NULL,
    category VARCHAR(30) DEFAULT NULL COMMENT '分类：色情/政治/广告/其他',
    level TINYINT DEFAULT 1 COMMENT '严重等级1-3',
    is_active TINYINT(1) DEFAULT 1,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_category (category),
    INDEX idx_active (is_active)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- 审核记录表
CREATE TABLE IF NOT EXISTS audit_logs (
    id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    content_type ENUM('post', 'comment', 'message', 'user_profile') NOT NULL,
    content_id INT UNSIGNED NOT NULL,
    user_id INT UNSIGNED NOT NULL,
    content_text TEXT DEFAULT NULL,
    audit_result ENUM('pass', 'reject', 'pending') DEFAULT 'pending',
    reject_reason VARCHAR(200) DEFAULT NULL,
    auditor_id INT UNSIGNED DEFAULT NULL COMMENT '审核员ID，NULL表示自动审核',
    audited_at DATETIME DEFAULT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_type_status (content_type, audit_result),
    INDEX idx_user (user_id),
    INDEX idx_created (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- 系统配置表
CREATE TABLE IF NOT EXISTS system_configs (
    id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    config_key VARCHAR(50) NOT NULL UNIQUE,
    config_value TEXT NOT NULL,
    description VARCHAR(200) DEFAULT NULL,
    updated_by INT UNSIGNED DEFAULT NULL,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

**API设计**:
```
GET  /api/admin/sensitive-words                 - 敏感词列表
POST /api/admin/sensitive-words                 - 添加敏感词
PUT  /api/admin/sensitive-words/:id             - 更新敏感词
DELETE /api/admin/sensitive-words/:id           - 删除敏感词
POST /api/admin/sensitive-words/batch-import    - 批量导入敏感词

GET  /api/admin/audit/queue?type=&status=       - 审核队列
PUT  /api/admin/audit/:id/approve               - 审核通过
PUT  /api/admin/audit/:id/reject                - 审核拒绝
GET  /api/admin/audit/logs                      - 审核历史
GET  /api/admin/audit/stats                     - 审核统计

GET  /api/admin/config                          - 系统配置列表
PUT  /api/admin/config/:key                     - 更新配置
```

  - 操作日志查看和导出
  - 异常操作告警

**数据库设计**:
```sql
-- 管理员表（扩展users表或独立表）
CREATE TABLE IF NOT EXISTS admin_users (
    id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    user_id INT UNSIGNED NOT NULL UNIQUE COMMENT '关联用户表',
    admin_role VARCHAR(20) NOT NULL DEFAULT 'operator' COMMENT '角色：super_admin/admin/operator/auditor/cs',
    permissions JSON DEFAULT NULL COMMENT '权限配置',
    is_active TINYINT(1) DEFAULT 1,
    last_login_at DATETIME DEFAULT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

-- 操作日志表
CREATE TABLE IF NOT EXISTS admin_operation_logs (
    id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    admin_id INT UNSIGNED NOT NULL,
    action VARCHAR(50) NOT NULL COMMENT '操作类型',
    target_type VARCHAR(30) DEFAULT NULL COMMENT '操作对象类型',
    target_id INT UNSIGNED DEFAULT NULL COMMENT '操作对象ID',
    detail JSON DEFAULT NULL COMMENT '操作详情',
    ip VARCHAR(50) DEFAULT NULL,
    user_agent VARCHAR(500) DEFAULT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_admin (admin_id),
    INDEX idx_action (action),
    INDEX idx_created (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

## 📝 技术建议

### 权限设计建议
```
超级管理员 (super_admin)
├── 所有权限

管理员 (admin)
├── 用户管理
├── 内容审核
├── 举报处理
├── 数据查看
└── 推送管理

运营 (operator)
├── 数据看板
├── 活动管理
├── 推送管理
└── 内容审核

客服 (cs)
├── 用户查看
├── 举报处理
└── 反馈处理

审核员 (auditor)
├── 内容审核
├── 认证审核
└── 举报处理
```

### 操作日志记录
所有管理员操作都应记录到操作日志表，包括：
- 登录/登出
- 用户封禁/解封
- 内容删除/恢复
- 认证审核
- 配置修改
- 数据导出

### 安全建议
1. 管理后台独立部署，限制访问IP
2. 管理员登录需要二次验证（验证码）
3. 敏感操作需要再次输入密码
4. 定期审计操作日志
5. 限制管理员权限范围
